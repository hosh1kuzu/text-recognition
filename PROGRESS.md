# TextRecog 当前进度

更新时间：2026-05-01

## 当前状态

- 当前方案已回到 PaddleOCR，优先保证识别质量。
- 当前分支：`codex/modern-flat-ui`
- 目标合并分支：`main`
- 当前构建产物：`dist/TextRecog/TextRecog.exe`

## 已完成

### OCR 与稳定性

- 保持启动时加载 OCR，避免把模型加载成本转移到首次识别。
- OCR 子进程增加异常上报，主进程收到错误后显示提示，不直接崩溃。
- 增加日志文件，便于排查截图、快捷键、OCR 子进程和结果窗口问题。
- 日志使用滚动上限，避免文件无限增长。

### 截图交互

- 支持多显示器截图：每个显示器显示独立遮罩，选区仍限制在单个显示器内。
- 支持右键和 Esc 取消截图。
- 修复右键取消时事件透传到底层应用的问题：现在会消费完整的右键按下与释放。
- 修复多次截图后可能崩溃或遮罩不出现的问题。

### UI 与图标

- 完成 Modern Flat 风格改造。
- 结果窗口、设置窗口、托盘菜单适配深色模式。
- 字体尺寸已调大，提升可读性。
- 新增应用图标资源，并更新打包后的 `TextRecog.exe` 图标。
- 截图遮罩增加「右键 / Esc 取消」提示。

### 系统集成

- 增加单例运行，重复启动不会再创建多个主进程。
- 设置中增加「开机启动」选项，默认关闭。
- 托盘菜单显示修复，深色模式下文字可见。

## 已验证

- `build.bat` 构建成功。
- 构建版可以正常启动 OCR 子进程。
- 多次截图未复现崩溃。
- 右键取消截图不会再提前关闭遮罩。
- 开机启动已人工验证正常。

## 后续可选方向

- 如果继续追求更小体积，可以单独实验 RapidOCR / ONNX Runtime。
- Windows OCR 已验证体积更小，但识别效果不满足当前默认方案要求。
